1.Overview

Author: Mayuri Pandey

Data can be found abundantly over the Internet, and there are databases available, which help in the storage and retrieval of those data and information. Nowadays, there is a rapid growth of location-specific devices that need geographical features and spatial data, and there exist several challenges to maintain such data. Hence in 1997 the Open Geospatial Consortium developed few specifications and set few standards for adding various spatial functionality to the database systems [1].

Therefore, in this regard, Geospatial Database or Spatial Data has been well-defined as collective sets of stored information which is used to depict any data which is related to or consists of information about a particular location on the Earth’s Surface. The main goal of this report is to understand the concept of geospatial databases in brief and to have a thorough understanding of various tools available, how these tools will help us to optimize data [2].

2.Geospatial Databases

Author: Mayuri Pandey

Geospatial database, defined as a group of different data sets stored in a common System file folder. The database can be generalized as containing geographic information of countries, cities, different administrative divisions, etc [2]. It mainly has data about the locations, geographic features, and the relationship between them, which are stored as coordinates and topology. It is structured in a way as to ensure maximum productivity from the database[2].

It follows ESRI standard format and acts as collective storage for various data types like Raster, Vector, Tables, etc. Another important feature is also the scalability, i.e, single to multiple editing can be achieved [2]. It acts as a physical store for the geographical information using a database stem where we can not only access but also work with the physical instances such as a collection of the data sets using SQL, it consists of a model that represents and manages the geographical information which supports the rules to manage spatial integrity and the tools to work with the attributes[2].

3.Vector and Raster data in Geospatial database

Author: Khushbu Navdiwala

Raster Data: Pixels that are assembled through rows or columns are made by the Raster Model. This data includes digital drone footage, space-station photos, or even digital information that will specifically be shown. There is 2 type of Raster Data:

  1. Discrete raster is easy to separate as data because they are made by themes, where one value may describe land cover, and the river can indicate another.
  2. Continuous rasters represent above/below sea level values. A good example of continuous raster data is digital elevation models(3d surface visualization)[17].

Vector Data: Vector data are made of points, lines, and polygons, all of which are used in maps to reflect real-world characteristics. All geometry types store details in x/y coordinates [17].

  1. Point data typically reflects small objects, such as tiny houses, pylons, important landmark features [17].
  2. Line data reflect characteristics such as transport (roads, cart paths, trails), rivers, cables, pipes [17].
  3. Polygon data reflects characteristics, such as land cover, area hydrography, large structures, boundaries of city/county/state/country[17].

5.Tools

Author: Mayuri Pandey

There exist various tools or software for analyzing and computing different patterns, that can assist the user to understand them more precisely. The functionality of each of them is listed below [4]:

Falconview: This application supports multiple platform mapping functions. It has the ability to showcase a wide range of maps including varied types of geographically referenced overlays. It lacks spatial analysis and also it has limited cartography and mapping tools. Data editing and management are quite difficult in this software[4][5].

GeoDa: It is free software that has the capability to conduct an analysis of spatial data, spatial modeling, etc. It mainly deals with geostatistics and as an introductory tool for beginners to work with spatial data. This tool is promising in various sectors such as health care, economic development, etc[4][6].

GrassGIS: This software is designed for the management of land and planning of the environment. It is free software that is valued for its various uses in sectors concerning analysis, manipulation of terrain, image processing, etc. GrassGIS supports 3D raster analysis[4][7].

ILWIS: It is open-source software for wide and easy access for users. Some of the basic areas of operation for this software are editing, digitization, and displaying geographic data. Further, the software also facilitates remote sensing with tools for image enhancement, classification, and band manipulation[4][8].

MapWindows: The software is not a complete software for utilization in the Geospatial data processing of the intended tasks such as identity features, map viewer, processing tools, and print layout. This software is free to use[4][9].

QGIS It is a geoinformation system, which enables users to not only examine and change the spatial data but also create and exchange pictorial maps. A plus point of QGIS is that it can handle shapefiles, dxf, coverages, PostGIS formats. Georeferencing of images is also possible with this software[4][10].

gvSIG: This application is desktop-oriented and has the ability to store, analyze, capture any geographically referenced information to resolve complicated issues. It is also counted as the base 3D visualization tool available having the most interactive user-friendly UI[4][11].

SagaGIS: Originally designed for analysis of terrains such as watershed extraction, hill shading, and visibility analysis. The tools facilitate operations with multiple windows to lay forward all the analysis such as maps, scattered plots, histograms, etc. It also provides methods related to geosciences to the geoscientific community[4][12].

uDig: This application is free to use having an intuitive user-friendly interface. It is a desktop-oriented application that is supported by all operating systems and also provides the framework on which analytical problems can be resolved[4][13].

8.Advantages and Disadvantages of Geospatial Databases

Author: Khushbu Navdiwala

Advantages [19]:

Every GIS user's search is to construct a more precise representation of the real world. For many applications, this technology will take you closer to the objective. The basic concept behind this technology is to make it more intuitive to work with GIS data and applications. Any of the features that make it so useful to use this technology [21]:

  1. Centralized data management: In the same geospatial database, you can store various spatial and tabular data formats. It has never been easier to handle and access data. Protection and access rights for multi-users can be handled down to the level of the individual feature class [21].
  2. Continuous Sets of features: Without tiles or other spatial partitions, it can handle large feature sets. Based on the necessary data, you can set the spatial domain for each geodatabase you can generate [21].
  3. More detailed editing of data: Validation rules will avoid several mistakes and recognize others. You can monitor interactions and relationships among features with validation rules and define appropriate values for attributes [21].
  4. Versioning: Versioning is enabled by multi-user geodatabases, which means that more than one person can modify the database simultaneously. Using a conflict resolution dialog, you can reconcile modifications to the same features or attributes by more than one user. When disputes are settled, the database's master version is modified. In addition to simplifying the process of editing, versioning can be used by organizations to handle alternative engineering [21].
  5. Using Geospatial technology, we can get some specific visual location information, it has the ability to make maps from given images, and also can be used for large projects which are connected with geography. It also supports efficiency, then easy to gather information by using different types of tools, giving high precision for any result, it estimates accurate presumptions and tests [19][22].
  6. A normal database system uses indexes; spatial databases use a spatial index to speed up the operations of the database [19][22].
  7. From a financing point of view, Geospatial is a one-off investment. It can be utilized in the longer term for further observation as long as the information is gathered. It chooses to keep vibrant after two decades of advent. In recent years, the usage of this technology continues to increase at an impressive speed as more and more users cherish the value which is derived over the years, the performance of the method/task has been boosted by the adoption of this technology [19][22].

For individuals looking to leverage the technology to its fullest potential, this is the greatest hurdle. Not understanding the significant applications makes it much harder for individuals to use the technology.
